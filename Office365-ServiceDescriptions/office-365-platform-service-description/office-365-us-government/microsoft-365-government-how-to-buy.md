# <a name="microsoft-365-government---how-to-buy"></a>Microsoft 365 정부 - 구입하는 방법

미국 공공 부문의 독특하고 진화하는 요구 사항에 대응하여 Microsoft는 미국 공공 부문 고객을 위한 특정 Microsoft 365 정부 환경을 조성했습니다. 이 서비스 설명에서는 환경에 대한 개요와 구매 방법에 대한 자세한 정보를 제공합니다. 다른 Microsoft 365 및 [Office 365 서비스 설명과](../../office-365-service-descriptions-technet-library.md)함께 이 서비스 설명을 읽는 것이 좋습니다.
  
## <a name="whats-microsoft-365-government"></a>Microsoft 365 정부는 무엇입니까?

Microsoft 365 정부는 통제되고 기밀되지 않은 정보를 보유하도록 후원하는 미국 정부 기관 및 계약자를 위해 맞춤화된 생산성, 보안 및 모빌리티 클라우드 소프트웨어 기능입니다. 가장 엄격한 규정 준수 요구 사항을 충족하는 고유한 환경을 통해 제공되는 Microsoft 365 정부는 Microsoft 상용 클라우드 엔터프라이즈 오퍼링의 기능과 기능을 가능한 한 가깝게 일치하는 미국 정부 고객을 위한 클라우드 제품입니다.
  
## <a name="whats-the-value-of-microsoft-365-government"></a>Microsoft 365 정부의 가치는 무엇인가?

Microsoft 365 정부는 기관이 규제 데이터 규정 준수 요구 사항을 효과적으로 관리할 수 있도록 지원함으로써 언제든지 모바일 생산성과 규정 준수와의 협력을 안전하게 제공함으로써 미국 정부 지도자와 근로자가 어디서나 생산성을 높일 수 있도록 지원합니다. 다음 표에서는 주요 정부 역할에 대한 예제를 제공합니다.<br><br>
 
| 정부 지도자들은 ... | 공무원은 ... | 현장 직원은 ... |
|:-----|:-----|:-----|
|커뮤니케이션 및 투명성 향상을 통한 대중의 신뢰 도증  <br/> |더 나은 협업을 통해 시민 서비스 개선 <br/> | 필요한 경우 필요한 정보 현장 담당자에 대한 모바일 액세스를 통해 서비스 품질 향상  |
|정책 및 예산 결정을 수립하는 데 도움이 되는 통찰력에 대한 액세스 개선  <br/> |기관 간 통신을 향상시킵니다.  <br/> |협업 및 지식 공유를 통해 문제를 신속하게 식별하고 해결할 수 있습니다.  <br/> |
|직원 및 시민, 사무실 또는 도로에서 정보를 얻고 연결하십시오.  <br/> |직원 유지율을 개선하기 위해 공무원과의 긴밀한 참여 창출  <br/> | 에이전시 위치 전반에 걸쳐 커뮤니티의 더 강한 감각을 구축  <br/> |

## <a name="what-makes-microsoft-365-government-different-from-microsoft-365-for-enterprise-offerings"></a>Microsoft 365 정부가 기업 용 Microsoft 365 차별화하는 이유는 무엇입니까?

Microsoft 365 정부는 네 가지 주요 영역에서 미국 정부 기관의 규정 준수 요구 사항을 해결하기 위해 독특하게 설계되었습니다.

**독점 커뮤니티.** 정부 데이터 센터는 미국 정부를 대신하여 정보를 보유하고 처리하도록 허가받은 정부 기관 또는 부서 또는 상업 회사에서만 사용할 수 있습니다.

**선별된 인력.** 조직의 고객 콘텐츠에 대한 액세스는 미국 시민이며 필요한 신원 조회를 통과한 Microsoft 직원을 선별한 것으로 제한됩니다.
타사 감사. 모든 인프라는 연방 기관이 운영 허가를 발급하는 데 사용하는 보안 평가 보고서 또는 증명 서신을 제공하는 공인 제3자 감사관에 의해 감사를 받았습니다. 

**콘텐츠 저장소.** 데이터는 준수 인프라를 통해 미국 대륙 내에 저장됩니다. 다음 표는 정부 환경 및 약정에 대한 Microsoft 365 요약합니다.<br><br>

| 제공 | 약속 |
|:----------|:----------------------------------|
|Microsoft 365 정부 GCC  <br/> |페드램프 하이, DFARS, CJIS, IRS 1075, DISA SRG L2 <br/> |
|Microsoft 365 정부 GCC 높음  <br/> |페드램프 하이, ITAR, DFARS, DISA SRG L4 컨트롤 <br/> |
|Microsoft 365 정부 DoD  <br/> |DISA SRG L5  <br/> |

Microsoft 보안 및 규정 준수 인증에 대한 자세한 내용은 [Microsoft 트러스트 센터를 방문하십시오. ](https://www.microsoft.com/trustcenter/default.aspx)  

## <a name="microsoft-365-government-eligibility-and-validation"></a>Microsoft 365 정부 자격 및 검증

Microsoft 365 정부는 통제된 정보를 보유하거나 처리하기 위해 후원하는 적격 정부 고객 및 비정부 조직이 이용할 수 있습니다. 이러한 두 범주의 고객에 대한 자격 기준은 Microsoft 정부 클라우드 에서 일관되습니다.

GCC 및 GCC 하이 오퍼링은 Microsoft 정부 클라우드를 받을 수 있는 모든 고객이 사용할 수 있으며, DoD 환경은 미국 국방부의 독점적 사용을 위한 것입니다. 서비스 가용성과 가격은 다르며, GCC FedRAMP 높음 또는 DoD가 통제한 기밀 정보(CUI)를 보유하지 않는 모든 고객에게 제공하는 영웅으로 남아 있습니다.

적격 한 정부 고객은 다음 하나로 정의됩니다.

- 미국 정부의 국, 사무실, 기관, 부서 또는 기타 법인으로 정의된 연방 기관
- 다음 중 하나로 정의된 상태/로컬 엔터티:
    - 미국의 주 또는 지방 정부의 모든 기관.
    - 모든 미국 카운티, 자치구, 연방, 도시, 지방 자치 단체, 마을, 마을, 특수 목적 지구 또는 고객의 국가의 법률에 의해 설립 및 고객의 국가의 관할권 및 지리적 경계 내에 위치한 기타 유사한 유형의 정부 도구
    - 컬럼비아 특별구, 푸에르토리코 연방, 괌, 아메리칸 사모아, 미국령 버진 아일랜드 및 북마리아나 제도
- 인도 부족으로서의 지위를 인정받아 미국 내무부로부터 자금 과 서비스를 받을 자격이 있는 연방 정부 인정 부족 법인으로 정의된 부족 법인또는 알래스카의 토착 마을 또는 알래스카 지역 원주민 법인으로 정의됨

다음과 같은 유형의 데이터를 보유하고 나열된 증거를 제공할 수 있는 비정부 조직은 Microsoft 정부 클라우드 유효성 검사 프로세스에 적합한 자격을 제공합니다.

규정의 적용을 받는 데이터가 있는 상업용 민간 법인입니다. 허용된 정부 데이터 유형은 다음과 같습니다.
- 무기의 국제 교통 (ITAR)
- 제어되지 않은 정보(CUI)
- 국방부 (국방부) 분류되지 않은 통제 핵 정보 (UCNI)
- 에너지부 (DoE) UCNI
- 형사사법정보(CJI)
- 국방부 영향 수준 데이터
- Microsoft 365 정부가 필요한 다른 유형의 데이터

국제 상업 법인은 미국 자회사를 통해 서비스를 구매해야 할 수 있지만 자격을 갖추게 될 수 있습니다.

위에 나열된 그룹 중 하나의 회원 자격 증명은 정부 Microsoft 365 제공에 액세스하려면 필요합니다.
